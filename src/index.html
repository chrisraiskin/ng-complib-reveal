why component libraries?

  • to achieve theme/style consistency
  • we wanted to distribute:
    • common theme components, like header and footer
    • that we could include into Angular template
      •
    • common services, e.g. IAM services
    • global styles
    • global assets

It was a cold winter of 2016 ..

  • Angular 4 just came out and
  • angular/cli still had no library support..

Our options in 2016

Essentially, two:

  1. a fairly complex one that
    • compiles the sources
    • inlines CSS and HTML
    • runs Rollup.js to build UMD* and ES5 modules
    or ..
  2. a relatively simple one that
    • publishes TypeScript sources, styles and assets to npm.


*UMD - Universal Module Definition for JavaScript modules

Which option

was the right one for us in 2016?

Considering:

  • our target audience
    • company-internal apps
  • we went with option #2
    • publish sources, styles and assets to npm

If we were writing for the OSS community
we'd likely had no choice but to go with option #1:
compiled sources, UMD, etc..

Pre ngV6 component library

  • Goals:
    • create angular project
    • containing sources, styles and assets
    • to be published as npm module to internal registry
    • also containing components allowing you
    • to test your library locally

Step 1: Create a new project

  • this will be
    • the wrapper project and
    • the test consumer for your library

                    
        $ ng new ng-complib --prefix sfg-complib

Step 2: create a new module

  • The library will reside in its own module


        $ ng g module component-library

Step 3: create library components

  • create Header component
                        
        $ ng g c component-library/header
  • create Footer component
                        
        $ ng g c component-library/footer
  • and so on

Step 4: export your library components

  • In your library module, export your components
component-library.module.ts


@NgModule({
  providers: [ /* Don't add the services here */ ],
  imports: [
    CommonModule
  ],
  declarations: [
    HeaderComponent,
    FooterComponent
  ],
  exports: [
    HeaderComponent,
    FooterComponent
  ]
})
export class ComponentLibraryModule {
  static forRoot() {
    return {
      ngModule: ComponentLibraryModule,
      providers: [
        // your provider services here
      ]
    };
  }
}
